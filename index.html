Implementation Notes

  1. This solution uses your provided DeepAR URL: https://sdk.developer.deepar.ai/effectTester/...
  2. The Switch Camera button should work if the iframe has camera access
  3. If the camera doesn't switch, it may be due to browser security restrictions
  4. For a production solution, you may need to:
// In your DeepAR iframe page, add this message listener: window.addEventListener('message', function(event) { // Verify the origin for security if (event.origin !== "https://your-parent-domain.com") return; var message = event.data; if (message.command === 'switchCamera') { // Call DeepAR switchCamera() function here if (typeof deepar !== 'undefined') { deepar.switchCamera(); } } }); // After switching camera, send status back function sendStatusToParent(status) { window.parent.postMessage({ type: 'cameraStatus', status: status, camera: deepar.getCurrentCameraName() // or your method to get current camera }, "https://your-parent-domain.com"); }

Note: Due to browser security policies, cross-origin iframe control may be limited unless both sites have proper CORS configuration.
